Analisis Klasifikasi Decision Tree: Dataset Play Tennis menggunakan KNIME#

Repositori ini berisi penjelasan mengenai proses pengolahan data menggunakan algoritma Decision Tree pada dataset ‘Play Tennis’ melalui tool KNIME Analytics Platform.

1. Persiapan Dataset#

Dataset yang digunakan berjumlah 14 baris data yang merepresentasikan kondisi cuaca harian. Atribut yang digunakan meliputi:

  • Outlook (Sunny, Overcast, Rain)

  • Temperature (Hot, Mild, Cool)

  • Humidity (High, Normal)

  • Wind (True, False)

  • Target Variable: Play Tennis (Yes/No)

2. Workflow KNIME#

Workflow dibangun secara sistematis dengan urutan sebagai berikut:

  1. Excel Reader: Mengambil data mentah.

  2. Column Filter: Menyaring kolom yang diperlukan.

  3. Table Partitioner: Membagi data menjadi dataset pelatihan (training) dan pengujian (testing).

  4. Decision Tree Learner: Melatih model pohon keputusan.

  5. Decision Tree Predictor: Melakukan prediksi pada data pengujian.

  6. Scorer: Menghitung akurasi model.

3. Konfigurasi Table Partitioner#

Pada tahap pembagian data, dilakukan pengaturan sebagai berikut:

  • Partitioning Type: Relative (%) sebesar 70%. Artinya 70% data digunakan untuk training dan 30% untuk testing.

  • Sampling Strategy: Stratified. Strategi ini dipilih untuk menjaga proporsi label ‘Yes’ dan ‘No’ tetap seimbang antara data training dan testing.

  • Group Column: Play Tennis.

Berdasarkan gambar hasil partisi, dari total 14 data, sebanyak 10 data masuk ke dalam dataset training.

4. Konfigurasi Decision Tree Learner#

Untuk membangun model pohon keputusan, node Learner diatur dengan:

  • Class Column: Play Tennis.

  • Quality Measure: Gain Ratio. Metode ini dipilih untuk menentukan atribut mana yang paling efektif menjadi pemecah (split) pada tiap node.

  • Pruning: Menggunakan Reduced Error Pruning untuk menghindari overfitting.

  • Min number records per node: 1.

5. Visualisasi dan Analisis Output (Tree View)#

Setelah model dijalankan, terbentuklah struktur pohon keputusan sebagai berikut:

  • Root Node (Akar Utama): Outlook. Atribut ini memiliki informasi paling kuat untuk membedakan target.

  • Cabang Overcast: Jika cuaca Mendung (Overcast), maka keputusannya adalah 100% Yes (n=3).

  • Cabang Sunny: Jika cuaca Cerah (Sunny), pada dataset training ini keputusannya adalah 100% No (n=2).

  • Cabang Rain: Jika cuaca Hujan (Rain), terdapat ketidakpastian (n=4, 75% Yes dan 25% No), sehingga membutuhkan atribut tambahan di level bawahnya untuk menentukan hasil akhir.

6. Kesimpulan#

Dengan menggunakan algoritma Decision Tree di KNIME, kita dapat secara otomatis menentukan aturan bisnis sederhana untuk memprediksi aktivitas. Penggunaan Gain Ratio membantu model menentukan bahwa Outlook adalah faktor terpenting dibandingkan suhu atau kelembapan dalam dataset ini.
